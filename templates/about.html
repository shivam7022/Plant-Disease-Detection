About The Project

LeafCare AI

A full-stack AI system for plant disease detection using deep learning

🏗️ System Architecture

🧠

Deep Learning Model

MobileNetV2 (transfer learning) + custom classification head trained on PlantVillage dataset

  • Input: 224×224 RGB images
  • 38-class softmax output
  • 97%+ validation accuracy
  • Adam optimizer · Categorical cross-entropy
⚙️

Backend Pipeline

Python-powered processing chain from image upload to prediction result

  • OpenCV: BGR→RGB, resize 224×224
  • NumPy: Normalization, array ops
  • TensorFlow: CNN model inference
  • Flask: REST API + templating
🗄️

Data Layer

SQLite database stores all predictions with metadata for history and analytics

  • SQLAlchemy ORM models
  • Predictions table + Disease info table
  • Timestamps, severity, confidence scores
  • REST JSON API at /api/predictions
🎨

Frontend

Modern responsive UI with drag-and-drop upload and animated results

  • Vanilla HTML, CSS, JavaScript
  • Dark green glassmorphism theme
  • Animated confidence bars
  • Mobile-responsive layout

📊 Dataset — PlantVillage

54,306 Total Images
38 Disease Classes
14 Plant Species
224×224 Image Resolution

The PlantVillage dataset (Hughes & Salathé, 2015) contains 54,306 labelled images of healthy and diseased plant leaves collected under controlled conditions. It is the standard benchmark for plant disease classification using deep learning.

🔬 CNN Model Architecture

Input Layer 224 × 224 × 3 RGB
↓
MobileNetV2 (Base) Pre-trained on ImageNet · Frozen layers
↓
Global Average Pooling 1280-dim feature vector
↓
Dropout (0.3) Regularization
↓
Dense (256, ReLU) Fully connected layer
↓
Output (38, Softmax) 38 disease class probabilities
